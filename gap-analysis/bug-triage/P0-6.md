# P0-6: Fix hardcoded `/en/` URL in email notifications

```yaml
- id: P0-6
  title: "Hardcoded `/en/` URL in email notifications"
  file: app/backend.server/models/user/invite.ts
  line: 53
  root_cause: >
    The `inviteURL` is generated using `ctx.fullUrl` (line 53) without considering the recipient's locale preference. This results in URLs that default to the English (`/en/`) interface, which is not appropriate for users with non-English locale preferences. The correct behavior is to dynamically include the user's locale in the URL or fall back to the system default locale if no preference is set.
  fix_complexity: Simple
  requires_spec: false
  requires_test: true
  fix_summary: "Derive the language prefix for the URL from the user's locale preference or the system default locale."
  antipattern: none
```
