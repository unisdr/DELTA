# P0-9: Fix `handleTransaction` sentinel string

```yaml
- id: P0-9
  title: "Fix `handleTransaction` sentinel string"
  file: app/backend.server/models/division.ts
  line: 76
  root_cause: >
    The `handleTransaction` function uses a string sentinel ("ERROR") to signal failure
    instead of throwing a typed Error. This bypasses TypeScript's control-flow analysis
    and makes error handling less robust and harder to enforce exhaustively. The correct
    approach is to throw a typed Error subclass, allowing callers to use `instanceof`
    checks for error handling.
  fix_complexity: Simple
  requires_spec: false
  requires_test: true
  fix_summary: "Replace the string sentinel with a typed Error subclass."
  antipattern: AP-06
```
